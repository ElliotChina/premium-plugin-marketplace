# 股票价值分析技能

## 技能概述

本技能提供全面的股票价值分析框架，生成结构化的投资分析报告。

### 核心原则

- **严格流程**：按照分析流程进行，不要跳过步骤，保证覆盖所有分析目标，确保分析的系统性和完整性
- **数据驱动**：所有结论必须基于数据分析，避免主观臆断，财务数据有延迟，应以最新披露为准，关键指标应交叉验证
- **来源标注**：报告中的所有数据必须标注数据来源（如 东方财富(akshare)、东方财富、同花顺、公司公告等），确保数据可追溯性
- **报告文件**：分析结果必须以报告文件形式输出，确保结构化和可读性，默认格式为 Markdown
- **风险提示**：报告中必须包含风险提示和免责声明

### 输出标准
- 13个分析目标的分析结论文档（按依赖关系顺序分析）
- 价值分析报告（必须先生成分析结论文档，再生成报告）

---

### 数据优先级

1. 业绩预告/业绩快报（如有）
2. 季度 TTM 数据（最新 4 个季度滚动）
3. 最新年报数据
4. 分析师预期（如需要预测数据）

## 分析流程详细说明

### 分析流程概述
1. **阶段 1：目标确认** - 通过提问确认分析的股票、市场类型和分析目标范围
2. **阶段 2：数据采集** - 采集所有基础数据，并进行数据校验
3. **阶段 3：数据加工** - 对基础数据加工处理，计算关键指标（如 TTM）
4. **阶段 4：数据分析** - 根据采集和加工的数据，按照 `references/` 目录下的 `*.md` 文件中的分析方法进行分析，得出结论
5. **阶段 5：生成报告** - 综合阶段4得出的所有分析结论，按照模板生成结构化的分析报告，并进行审查确保内容一致性和完整性


### 阶段 1：目标确认

#### 需要确认的信息
通过向用户提问，确认以下信息：
1. **股票名称/代码**：确认用户需要分析的股票名称或代码
2. **市场类型**：确认股票所在的市场类型（A 股/港股/美股），确保分析方法适用
3. **分析目标**：向用户说明可分析的目标范围，并让用户选择需要分析的目标，确保分析内容符合用户需求

### 阶段 2：数据采集

#### 数据采集规范
1. 全面分析，直接使用全量采集脚本（默认方式）
2. 如果是对部分目标分析，则先读取分析目标的 `references/*.md` 分析方法文档，了解需要采集的数据需求，使用指定类型采集脚本
3. 数据保存路径为 `$DATA_DIR`
4. 将采集到的数据分别保存为CSV文件，文件命名规则为 `[数据类型]_[数据源]_[接口名].csv`。如`资产负债表_同花顺_stock_financial_debt_new_ths.csv`、`基础信息_巨潮资讯_stock_profile_cninfo.csv`等

#### 全量采集（快速采集）

使用 `collect_fundamental.py` 脚本一键采集所有16种数据类型（56个接口）：

```bash
# 基础用法：采集所有类型数据
python scripts/collect_fundamental.py --symbol 000001

# 指定输出目录
python scripts/collect_fundamental.py --symbol 000001 --output $DATA_DIR

# 排除某些数据类型（如市场热度和新闻研报）
python scripts/collect_fundamental.py --symbol 000001 --exclude 市场热度 新闻研报

# 指定历史起止日期（如历史行情数据）
python scripts/collect_fundamental.py --symbol 000001 \
  --start-date 20200101 --end-date 20231231
```

**支持的数据类型（16种）**：
- 基础信息、公司主营、历史行情、同行比较、市场热度
- 个股估值、新闻研报、业绩快报、资产负债表、利润表
- 现金流量表、财务指标、股东户数、十大股东、持股变动、分红配送

**输出结果**：
- 每个接口生成独立的 CSV 文件，命名格式：`[数据类型]_[数据源]_[接口名].csv`
- 例如：`基础信息_巨潮资讯_stock_profile_cninfo.csv`、`财务指标_同花顺_stock_financial_abstract_new_ths.csv`

#### 数据源

优先从数据接口获取数据，其次通过网络搜索工具获取数据，必须保证分析的准确性和时效性

**优先级别**：
1. 数据接口（如 akshare），如果接口调用失败可以尝试使用备选接口
2. 网络搜索工具（如东方财富、同花顺、雪球等）

| 数据类型 | 第一优先级（主数据源） | 第二优先级（备选） | 第三优先级（兜底） |
|----------|----------------------|-------------------|-------------------|
| **财务数据**（三张报表） | akshare 接口（最新季报/年报） | 公司官网投资者关系 | 手动输入/网络搜索 |
| **行情数据**（股价/市值） | akshare 实时行情 | 交易所官网 | 财经网站（新浪/腾讯） |
| **TTM 指标** | **优先使用数据源 TTM 值**（如净利润 TTM、营业收入 TTM） | 基于单季度数据计算 TTM | 手动计算 |
| **估值数据**（PE/PB/PS） | **PE/PS 必须用 TTM 版本**（优先数据源 TTM 值，否则自己计算） | - | - |
| **股东信息**（十大股东） | akshare 季报持仓 | 公司年报/公告 | 财经网站 |
| **分红数据** | akshare 分红配送 | 公司官网投资者关系 | 手动录入 |
| **行业数据** | akshare 行业板块 | 国家统计局 | 行业研究报告 |
| **业绩预告/快报** | akshare 业绩公告 | 交易所公告 | 财经新闻 |

### 阶段 3：数据加工

#### 数据校验

##### 数据校验步骤
1. 读取采集到的数据文件，检查数据完整性（是否有缺失值、是否有异常值）
2. 交叉验证关键指标（如净利润、营业收入等）与其他数据源的数据是否一致
3. 对于财务数据，手动计算滚动 12 个月（TTM）指标，验证与季度报表数据的一致性
4. 发现数据问题时，记录问题类型和影响范围，并回到数据采集阶段，尝试使用备选数据源进行补充或替代

#### 数据计算

1. 读取分析目标的 `references/*.md` 文档，了解需要计算的数据和方法
2. **TTM 计算**：按"TTM 计算与获取"章节的优先级和规则处理
3. 数据保存到 `$DATA_DIR`，文件命名：`[数据类型]_[数据源]_[数据说明].csv`

##### TTM 计算与获取

**TTM（Trailing Twelve Months）= 最近 12 个月滚动数据**

**需要 TTM 的指标**：
| 指标类别 | 具体指标 | 用途 |
|---------|---------|------|
| **盈利指标** | 净利润(TTM)、扣非净利润(TTM)、营业收入(TTM) | 计算 PE、PS、PEG |
| **利润率指标** | 毛利率(TTM)、净利率(TTM)、ROE(TTM) | 财务质量分析 |
| **现金流指标** | 经营现金流净额(TTM)、自由现金流(TTM) | 现金流分析、估值 |
| **估值指标** | PE(TTM)、PS(TTM)、股息率(TTM) | 估值分析核心 |

**TTM 数据获取优先级**：
1. **优先使用数据源提供的 TTM 值**（效率最高、准确性好）：
   - akshare 等数据源通常直接提供 `净利润(TTM)`、`营业收入(TTM)`、`PE(TTM)`、`PS(TTM)` 等字段
   - 这些值已经过数据源的计算和校验，可直接使用

2. **数据源无 TTM 时，手动计算**：
   - **TTM = 最近季度 + 前三个季度**
   - 例如：2024Q3 TTM = 2024Q3 + 2024Q2 + 2024Q1 + 2023Q4
   - **PE(TTM) = 当前市值 ÷ 净利润(TTM)**
   - **PS(TTM) = 当前市值 ÷ 营业收入(TTM)**

**⚠️ 特别注意**：
- **PE/PS 必须用 TTM 版本**：数据源的 `PE` 通常是静态PE（基于上年年报），不符合本技能要求
- **必须明确标识 TTM**：查看数据字段是否包含 `(TTM)`、`滚动12个月`、`trailing` 等关键字
- **业绩预告/快报优先**：如果有最新业绩预告/快报，应优先采用，修正 TTM 计算

### 阶段 4：数据分析

#### 分析步骤

1. 分别读取分析目标的 `references/*.md` 分析方法文档，了解分析方法和判断标准
2. 读取 `$DATA_DIR` 中的数据文件，了解所需关键指标
3. 按照"分析逻辑与判断标准"进行评估
4. 保存分析结论到 `$ANALYSIS_DIR` 目录，文件命名为`[章节编号]_[章节名称]_分析结论.md`

**分析原则（重要）**
- 每个目标分析开始前，都必须读取对应的 `references/` 目录下的 `*.md` 文件，学习分析方法和判断标准
- 每个分析目标分析都严格按照完整流程进行分析：数据采集、数据校验、数据计算、数据分析、结论输出
- 必须记录数据采集结果和每个分析目标的分析结果，保存为 csv 或 Markdown 文件，确保分析过程的可追溯性和结果的结构化
- 每个目标的分析开始和完成后，必须向用户及时反馈进度提示：
    - 每个目标分析开始前，向用户输出该目标的分析内容和预期结果
    - 每个目标分析完成后，向用户输出该目标的分析结论

### 阶段 5：生成报告

**重要**
- 生成报告前必须保证所有分析目标的分析结论都已完成，并且已经保存到 `$ANALYSIS_DIR`

#### 生成报告流程：
1. **读取子目标分析结论**：从 `$ANALYSIS_DIR` 目录下读取各章节分析结论
2. **参照模板生成**：按照 `templates/report_template.md` 的结构组织内容，删除未分析章节，调整章节顺序以确保逻辑流畅
3. **添加免责声明**：报告末尾必须包含免责声明
4. **保存报告**：将最终报告保存到 `$OUTPUT_DIR` 目录下
5. **报告审查**：读取报告文件，按照要求进行审查

#### 审查要求
- **一致性**：审查报告中的数据、结论与分析文件一致
- **完整性**：确保报告内容完整、覆盖用户要求的分析所有目标。符合 `references/` 目录下的 `*.md` 文件中描述的分析内容要求
- **数据缺失**：严禁出现占位符、空数据、代码片段等。对于数据不完整或分析不确定的部分，提供明确提示
- **来源标注**：报告中所有数据必须标注数据来源（如 akshare、东方财富、同花顺、公司公告、交易所官网等），确保每个关键指标都可追溯到具体数据源
  - **直接采集数据**：标注数据源
    - 数据接口数据：标注接口名称（如 akshare）
    - 网络搜索数据：标注网站名称（如东方财富、同花顺、雪球）
    - 公司公告数据：标注公告类型和日期（如 2024 年三季报、业绩预告）
  - **计算得出数据**：标注"根据 xxx 数据计算得出"
    - TTM 数据：标注"根据最近 4 个季度数据计算得出"或"根据 akshare TTM 值"
    - 估值指标：标注"根据 akshare 数据计算得出"或"根据 akshare TTM 值"
    - 其他计算指标：标注计算依据的数据来源

#### 报告格式选择
- **Markdown**：简洁、易编辑、版本控制友好

#### 报告保存位置
- Markdown 格式：`$REPORT_PATH`

## 边界情况处理

### 数据缺失

- **股票停牌**：识别停牌状态，使用停牌前最新数据，提示时效性风险
- **新股上市**：历史数据不足时，使用招股书数据 + 同行对比，给出谨慎提示
- **亏损公司**：PE 估值失效，改用 PS、DCF 等替代方法

### 特殊市场

| 市场 | 注意事项 |
|------|---------|
| **港股** | 使用 `.HK` 代码格式，关注港股通持仓、汇率风险 |
| **美股** | 关注自由现金流、股票回购、FDA 审批等特有因素 |
